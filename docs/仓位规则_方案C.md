# 方案C仓位规则（V13正式版）

## 核心规则

### 1. 双限仓位
- **单票上限** = max_pos_pct%（弱春/混沌=15%）
- **单票软上限** = max_pos_pct% + position_tolerance%（默认15%+5%=20%）
- **总仓位上限** = max_total_pct%（弱春=35%，混沌=20%）

### 2. 买入逻辑
单笔买入金额 = min(总资金×max_pos_pct%, 当前现金)
受总仓位上限约束：超过max_total_pct%时不再开新仓

### 3. 回撤保护
从峰值回撤超过15%，自动将总仓上限降低25%（最低10%）

## 被动超限 vs 主动超限规则

### ✅ 被动超限（不违规，仅提醒）
**场景：** 股价上涨导致单票占比超过软上限
**处理：**
- 标记为"⚠️ 关注"提醒
- **不触发强制降仓**
- 不阻止后续其他操作

### 🔴 主动超限（拦截）
**场景：** 买入操作后单票占比超过软上限
**处理：**
- **拦截买入操作**（系统检查：当前市值+本次买入 ≤ 总资金×软上限）
- 不拦截卖出操作

### 🟢 软上限内（合规）
单票占比在 max_pos_pct% ~ 软上限 之间 → 正常持有，允许小幅补仓
（偏差值的用途就是覆盖A股100股一手的整数交易规则）

### ⚠️ 超过软上限
- 被动上涨导致 → 标记高亮提醒，不强制卖出
- 主动买入导致 → 拦截
- 超过总仓位上限 → 超额部分不能再开新仓
