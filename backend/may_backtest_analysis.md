## 二、核心问题深度分析

### 问题1: 混沌期13笔全亏 — 买入信号在混沌期完全失效

**数据表现**:
- 2023年7~8月的13笔交易全部亏损，平均每笔亏损约 **-7.35%**
- 买入日期集中在 2023-07-05 ~ 2023-07-20，此时市场处于 chaos 或 autumn
- 持仓天数极短（6~44天），多数在 T1/T2 止损中出局

**根因诊断**:
1. **评分引擎的 momentum 信号在混沌期是"假信号"** — 动量评分在震荡市场中天然追逐追高买入，但混沌期趋势持续性差，买入后随即反转
2. **V11参数下混沌期采用 buy_line=75，实际这个阈值筛掉的仍然是假信号** — 更关键的问题是混沌期根本不应该使用 momentum 轨道
3. **止损参数在混沌期与市场节奏冲突** — T1=10%/T2=8%的止损在趋势市场中合理，但在震荡混沌期中，股价的正常摆动就会触发止损

**建议方案**:

**方案A（推荐）: 混沌期不交易 — 最有纪律性**

```
chaos参数更新:
  buy_line=85          ← 提到极高，几乎不会触发
  max_hold=10          ← 缩短持有
  t1_stop=8            ← 收窄止损减少损失  
  t2_stop=5
  t2_enabled=False     ← 明确关闭T2
  实际效果: 混沌期几乎不出信号，资金空仓等待
```

估算影响: 混沌期平均占全年25-40%的时间，空仓可避免2023年13笔全亏的-95%回撤，但也会错过2024年9月混沌→春天转换期的底部买入机会。权衡后**利大于弊**。

**方案B（保守）: 混沌期降低买入线而非关闭**

```
chaos: buy_line=70  (降低买入门槛)
       但采用 B轨(均值回归)而非A轨(动量)
```

这意味着混沌期不追高，而是寻找超跌股。需要验证B轨在混沌期的胜率。

**方案C（激进）: 保留当前逻辑但增加混沌期短期过滤器**

即用 short_term_filter 对混沌期的买入信号做二次筛选。

**关键矛盾**：

**结论**: **混沌期T1=10%/T2=8%的参数本身没有大问题**，问题是 **混沌期不应该使用 momentum 评分来触发买入**。如果在混沌期开始交易，这个止损参数是合理的。核心矛盾在于"选时"而非"选股"。

**优化方向**:

| 场景 | 当前参数 | 建议调整 | 理由 |
|------|---------|---------|------|
| 混沌期(如开启交易) | T1=10%/T2=8% | T1=12%/T2=9% | 混沌期振幅大，放宽止损 |
| 混沌期不交易 | — | 无需设置 | 最简单有效 |

### 问题5: 短期模型作为混沌期辅助过滤器

**可行性分析**:

short_term_filter 的核心评估逻辑：
- 资金惯性(30%) — 买入后资金持续流入
- 动量衰减(25%) — 是否跌破均线
- 量能确认(25%) — 量比是否正常
- 超买风险(20%) — RSI和换手率

**当前评估**: short_term_filter 在混沌期作为辅助过滤器**效果有限**，原因：

1. **过滤器设计定位是"买入后持股1~5日内跟踪"**，不是"买入前预筛选"
2. 混沌期资金流向本身就缺乏持续性，资金惯性评分可能不稳定
3. 需要记住的是—**2023年混沌期的13笔交易平均持仓6~44天**，短期过滤器评估5天无法判断长线走势

**更适合的用法**: 不做买入前筛选，而是作为**买入后第3/5天的风控检查点**：

```
买入后第3天调用 short_term_filter:
  - 如果 composite_score < 35 (exit信号)
    - 混沌期: 立即减半仓
    - 非混沌期: 观察2天
  - 如果 composite_score < 50 (reduce信号)
    - 混沌期: 降低仓位到1/3
```

**替代方案**: 在混沌期改用 B轨(均值回归)评分，而非在A轨(动量)上叠加过滤器。B轨天然适合混沌震荡市。
