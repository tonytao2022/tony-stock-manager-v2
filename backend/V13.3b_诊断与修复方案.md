# V13.3b 评分机制失效 — 全面诊断与修复方案

## 时间：2026-07-17

## 第一部分：根因诊断

### 【根本问题】technical_indicator 表的 MA 字段自 6/23 起全部为 0

| 日期 | MA正常分配数 | 出问题总数 |
|:---:|:----------:|:---------:|
| 6/16 | 688只 ✅ | 0 |
| 6/17 | 343只 | 5157只 = 0 |
| 6/18 | 342只 | 5157只 = 0 |
| 6/23 ~ 7/14 | **0只** | **5514只全部 = 0** |

**影响时间线：**
- 6/16 最后一批正常MA数据
- 6/17 ~ 6/18 开始恶化（监控池从342→5500+导致stk_factor超时）
- 6/23 ~ 7/14 技术指标管道**持续运行但全部写入0**
- 7/15 ~ 7/16 V13.3b上线，但MA兜底机制7/17凌晨才加入

**导致的具体问题链：**

1. `trend_score`（30%权重）在 `track_momentum` 中有一段根据 `MA20/MA60` 关系调整趋势分的逻辑（L397-401），但只在ma20>0时才运行——MA=0时代码取0，分支条件失效
2. 价格验证（跌破MA20打折）在V13.3b之前完全没有——因为没有MA数据
3. 空头排列检测在V13.3b之前完全没有
4. 缠论结构（bi_lever/buy_sell_point）本身不依赖MA，继续正常工作但给出"虚假"的买入信号

### 【结构性缺陷1】7/2大跌日评分反升

**芯源微 7/2：408→360 (-11.8%)**
- 评分从 78.1 → **91.5**（上升！）
- trend_score: 55 → **100**（缠论认为形成"类一买"）
- momentum_score: 95 → 80.2（已含大跌影响但权重25%）
- structure_score: 46.3 → 82.3（缠论结构向好）
- penalty: 0（MA=0，价格验证没生效）

**总结：缠论结构在突发事件（大跌）后认为形成了"底部"结构，反给高分**

### 【结构性缺陷2】缠论结构在扩展后产生大量虚假信号

监控池从342→5500+后，缠论结构对大量趋势下跌的股票产生了"类一买/类二买"信号。因为缠论的"笔"算法识别的是局部反转结构，不考虑绝对趋势方向或基本面恶化。

### 【结构性缺陷3】评分引擎缺乏"趋势方向"维度

当前5个子因子权重：trend(30%) + momentum(25%) + mf(15%) + margin(10%) + pos(10%) + structure(10%)

- trend_score 不是"趋势方向"，而是"缠论结构质量"
- momentum_score 是短期动量（5日涨跌幅+成交量放大），这个是真的反映趋势方向的
- 没有**均线方向、长期均线斜率**这种趋势确认指标

### 【结构性缺陷4】校准机制的扭曲效应

百分位映射校准对暴跌股有 "拉回效应"：
- 当大量股票同时暴跌时，校准分布整体下移
- 芯源微在7/10的composite=77.8被校准到100（全市场前5%）
- 7/15的58.5被校准到93.7（其他股票也跌了，校准分相对保持高位）

## 第二部分：修复方案

### P0：修复 technical_indicator 数据源（源头修复）

**方案A：从K线自算MA（已在V13.3b实现）**
✅ 生效中
✅ 不依赖Tushare接口
✅ 计算效率高（O(n)）
❌ 只能算基础MA，不能算MACD/BOLL/RSI
❌ 只能用于引擎内部，不会修复数据库

**方案B：修复close_pipeline.sh的stk_factor调用（推荐）**
根因：监控池5500+导致stk_factor查询超时120秒

修复方案：
1. 对5500+监控池分批查询（每批1000只）
2. 增加超时时间到300秒
3. 对超时的批次自动降级为从K线自算

**方案C：数据回填脚本**
- 一次性跑回填，用K线自算填补6/18~7/14所有缺失的MA
- 脚本位置：`/root/stock-system-v2/backend/scripts/fill_technical_indicator.py`

**选择：方案B+C同时进行，方案A作为长期兜底**

### P1：评分架构加固

**1. 增加趋势方向检查（长期）**
```
# 新增：20日均线方向 + 60日均线方向
ma20_trend = (ma20_today - ma20_20days_ago) / ma20_20days_ago
ma60_trend = (ma60_today - ma60_60days_ago) / ma60_60days_ago

if ma20_trend < -0.02:  # 均线向下趋势明显
    trend_score *= 0.7  # 趋势分打折
```

**2. 暴跌自检（短期）**
```
# 单日跌幅>5% + 评分反而上升 → 强制扣分
if daily_drop < -5% and score_increased:
    penalty += 15
```

**3. 缠论信号验证**
```
# 一买信号 + 价格在MA20以下 + 趋势分<50 → 无效信号
if bs == 'buy_spot_1' and close < ma20 and trend < 50:
    bs = 'dubious_buy'  # 不确认
```
